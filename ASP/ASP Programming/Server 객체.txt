Server 객체는 서버 측에서 사용할 수 있는 각종 메소드와 속성을 활용할 수 있도록
지원해 주는 객체이고, 이러한 메소드와 속성의 대부분은 일종의 유틸리티와 같은 기능을
수행한다.

Server 객체는 사용자가 필요로하는 서버 컴포넌트의 인스턴스를 생성할 수 있도록 지원하는
기능을 가지고 있다.

Server 객체는 CreateObject 메소드를 이용하여 ASP 환경 내에 객체를 생성한 후에 외부에 존재하는 컴포넌트나 스크립트 객체 등이 가지고 있는 메소드나 속성을 이용할 수 있도록 지원한다.
또한 Server 객체는 웹사이트 구축에 필요한 부수적인 기능, 즉 스크립트의 실행시간 제어,
파일 매핑 , 인코딩을 할 수 있는 메소드나 속성을 제공한다.


[컬렉션, 메소드 , 이벤트]

[속성]

ScriptTimeOut  : 에러 발생 전까지의 스크립트의 실행 시간 설정

[메소드]

HTMLEncode : 문자열을 HTML 형식으로 코드화

URLEncode : 문자열을 읽어 URL 문자열의 형식으로 코드화

MapPath : 가상의 경로를 서버에 있는 물리적인 경로로 변환

CreateObject : 사용할 서버 컴포넌트 객체의 인스턴스를 생성

Execute : 실행 중에 다른 페이지를 실행하여 결과를 현재 페이지에 포함 시킴(ASP 3.0)

Transfer : 실행 중에 다른 페이지로 실행을 옮김 (ASP 3.0)



[ScriptTimeOut 속성]

ScriptTimeOut 속성은 스크립트의 실행 시간이 너무 길어질 경우를 대비하여
ASP 어플리케이션의 스크립트 실행 시간을 설정하고 제한하는 기능을 한다.
서버 컴포넌트가 처리되고 있는 동안에는 이 속성이 갖고 있는 스크립트의
최대 실행 시간에 적용되지 않는다.
지정 단위는 초(Seconds)이며, 스크립트의 기본 실행 시간은 90초이다.
지정된 시간을 경과하면 해당 스크립트는 실행이 완료되지 않았더라도 서버의 
자원을 절약하기 위하여 오류 메시지와 함께 즉시 종료된다.

<% Server.ScriptTimeOut = 120 '120초로 설정 %>


[HTMLEncode 메소드]

HTMLEncode는 브라우저에 <Br>과 같은 HTML 문자 자체(태그)를 내용으로 출력시키고자
할 때 사용하는 메소드이다.

Server.HTMLEncode("문장")

[URLEncdoe 메소드]

GET 방식이나 하이퍼링크로 다른 페이지에 연결하여 데이터를 넘겨줄 때,
전송되는 데이터 값은 인코딩하여 전송한다. 특히 한글이나 기호 문자들은 %와
ANSI 코드 16진수 값으로 전달된다.
만약 퍼센트를 나타내는 %기호나 더하기 기호인 + 등을 넘겨주어야 할 경우에 URLEncode 메소드를
사용해야 한다. 즉, URL 인코딩은 URL에서 사용할 수 없는 특수문자를 전송할 때 그에 
대응하는 코드로 넘겨주거나 URLEncode 메소드를 사용해야 한다.

[특수문자의 URL 인코딩 전과후]

전 / 후

공백(Space) / +
" / %22
? / %3F
% / %25
+ / %2B
& / %26
= / %3D


[MapPath 메소드]

MapPath 메소드는 논리적인 경로를 서버의 물리적인 경로로 바꾸어지눈 기능을 한다.
가상 디렉토리를 저장장치의 물리적인 경로로 바꾸어 준다.

현위치 : Server.MapPath(".")
상위 폴더 :  Server.MapPath("..")
웹 루트 폴더 : Server.MapPath("/")
현 프로그램의 URL : Server.MapPath(Request.ServerVariables("URL"))

[CreateObject 메소드]

CreateObject는 서버 내에 등록되어 있는 OLE / ActiveX 컴포넌트를 사용할 수 있도록
지원하여 ASP가 현재 갖고 있지 못한 기능을 구현하도록 만들어 주는 메소드이다.

CreateObject 메소드를 사용하는 일반 형식은 다음과 같다

Set 객체 인스턴트명 = Server.CreateObject("클래스.컴포넌트")

예를 들어, 데이터베이스를 사용할 경우에, 다음과 같이 작성하며,

Set objDb = Server.CreateObject("ADODO.Connection")

스크립트 객체인 경우에는 다음과 같이 작성한다.

Set objfile = Server.CreateObject("Scripting.FileSystemObject")


[Execute 메소드]

ASP 3.0(웹 서버 II 5.0)에서 제공하는 메소드로서 하나의 웹페이지를 실행하다가
실행 중에 다른 웹페이지를 읽어오는 방법을 제공한다.
또한 이와 유사한 메소드인 Transfer 메소드가 있다.

Execute 메소드는 마치 함수를 사용하듯이 명시된 다른 웹페이지를 호출하고
그 페이지의 실행이 완료되면 다시 원래의 페이지로 되돌아와 다음 문장을 계속
수행하는 메소드이다.

Server.Execute "호출할 다른 페이지"

한편, Transfer 메소드는 하나의 웹페이지를 실행하다가 다른 페이지를 호출하면
그 페이지에 제어권을 넘겨주고, 실행이 끝나면 그것으로 페이지가 종료된다.
