ASP가 제공하는 객체의 구성 요소는 컬렉션(Collection) , 속성(Property) , 메소드(Method) 및 이벤트 (Event) 등이 있다.

[구성요소]

1. 컬렉션(Collections)

컬렉션은 문자열 ,숫자 , 객체 등을 저장하는 자료 구조이며 배열과 유사하지만 배열이 숫자를
인덱스로 사용하는 반면에 , 컬렉션은 문자열을 인덱스(키 값)로 사용한다. 컬렉션은 배열이 제공하는 기능 외에도
항목을 쉽게 찾거나 저장할 수 있다는 이점을 가지고 있으므로 유연한 처리를 지원해 준다.


2. 속성(Property)

속성은 객체가 가지는 값 또는 성질이라고 할 수 있는데, 객체가 가지는 내부의 변수나 하위 객체가 이러한 속성에 해당한다.
응용프로그램을 통해 객체의 속성을 참조하거나 속성을 변경할 수 있다.

3. 메소드(Method)

메소드는 객체가 수행하는 동작을 말하며, 보통 실행 명령으로 구성된다.
메소드는 객체가 가지는 함수에 해당하는 개념이다.

4. 이벤트(Event)

이벤트는 객체의 속성을 바꾸거나 메소드를 동작시키기 위해서 필요로 하는 동기나 자극을 말한다.
예를 들어, 마우스의 클릭 , 더블클릭 , 드래그 등 사용자의 움직임이 이벤트가 되며 , 
또 시스템 상태의 변경이나 타이머의 동작에 의해서도 이벤트가 발생할 수 있다. 
우리가 작성할 동적인 웹 응용 프로그램은 이러한 이벤트가 발생되었을 때, 
동작할 수 있는 메소드나 함수를 작성하는 것이라고 할 수 있다.



[객체 모델]

객체 모델은 여러 종류의 객체와 이들간의 관계를 정의해 놓은 것이다. ASP가
제공하는 객체는 클라언트와 서버간의 통신 작업을 주로 지원하여, 객체의 메소드와
속성은 스크립트를 위한 유틸리티 함수들을 제공해 준다. ASP는 Request 객체, Response 객체
Applictaion 객체 , Session 객체 , Server 객체 등을 제공하고 있다.

1. Reqeust 객체

Request 객체는 HTTP 요청 시에 클라이언트(브라우저)에서 서버로 전송하는 데이터를 저장하는 객체이다.
그러므로 이 객체를 이용하면 클라이언트가 서버로 보내는 값을 파악할 수 있고, 이에 따르는 적절한 처리 작업을
수행할 수 있다.

2. Response 객체

Response 객체는 서버에서 클라이언트로 전송하는 데이터를 저장하는 객체이다.
그러므로 Request 객체와는 반대되는 개념의 역할을 수행한다고 할 수 있다.

3.Application 객체

Application 객체는 어플리케이션을 사용하는 모든 사용자의 대하여 공통적인 정보를 저장하는 객체이다.
예를 들어 현재까지 특정 어플리케이션을 방문한 사용자의 수 등을 파악할 수 있다. Application 객체의 내용은
웹 서버의 특정 가상 디렉토리 및 이의 하위 디렉토리 모두에 대해서도 적용된다.

4. Session 객체

Session 객체는 개별 사용자에 대한 정보를 저장하는 객체이다. Application 객체가 다수의
사용자에게 임의의 어플리케이션에 대한 정보를 공유할 수 있도록 지원해 주는 객체라고 한다면,
Session 객체는 동일 어플리케이션내의 페이지 간에 데이터를 공유할 수 있도록 지원해 주는 객채가 된다.
그러므로 임의의 사용자에 대하여 Application 객체와 Session 객체의 내용을 모두 적용시킬 수가 있다.

5.Server 객체

앞에서 살펴 본 객체들은 ASP가 제공하고 있는 객체이므로 별도의 정의 과정이나 생성 작업을 거치지 않고서도
바로 사용할 수 있는 객체이다. 그러나 Server 객체는 외부에 존재하는 컴포넌트나 스크립트 객체를 사용할 수 있도록
지원하는 객체이다. 또한 Server 객체는 웹사이트 구축에 필요한 부수적인 기능을 발휘할 수 있도록 많은
메소드와 속성을 제공한다.
