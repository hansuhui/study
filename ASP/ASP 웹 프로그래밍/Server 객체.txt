Server 객체는 ASP에서 제공하지 않는 객체를 생성하여 활용할 수 있도록 고안된 객체이다.

[속성]

ScriptTimeout : 스크립트의 실행시간을 설정

[메소드]

CreateObject : 객체 , 서버 컴포넌트의 인스턴스 생성

HTML Encode : 지정된 문자열 HTMl 인코딩

URL Encode : Escape 문자를 포함한 문자열에 URL 인코딩

MapPath : 가상경로를 물리적 경로로 변환



[ScriptTimeout 속성]

이 속성은 스크립트의 실행시간이 너무 길어질 경우를 위한 것으로 즉, 어떤 ASP의 페이지에서
프로그래밍 실수로 어떤 루프가 무한대로 도는 것을 방지하기 위한 것이다. 
즉,어떤 페이지가 일정시간이 지나면 스크립트가 강제로 실행을 멈출 수 있도록 세팅하는 것으로,
기본적으로는 90초로 설정되어 있으며, 그 이상 실행되어야 할 명백한 이유가 있다면 , 연장할 수 있다.
물론 그 이하도 가능하다. 다음은 스크립트 종료시간을 150초로 설정한 예이다

<% Server.ScriptTimeout = 150 %>


[HTMLEncode 메소드]

이 메소드는 문자에서 말해주는 것과 같이 HTML 코드 자체를 HTML 페이지에서 사용하는 경우 사용된다.
사용자의 브라우저에 <HTML>과 같은 문자를 보여주기 위해서는 HTML 문서에 &it;HTML&ht; 라고 코딩하는 것과 마찬가지로
태그용 문자나 혹은 특수문자는 그에 대응 되는 코드를 사용한 경험이 있을 것이다. ASP 에서도 이러한 원칙이
그대로 적용되지만, 그렇다고 해서 일일이 이것들을 검토해 주는 것은 복잡한 일이므로 HTMlEncode라는 메소드를
통하여 자동으로 변환할 수가 있는 것이다.


[URLEncode 메소드]

Request 객체에서 Get 방식을 이용하여, 폼에 입력한 내용을 전송하면,  URL에 데이터가
꼬리에 붙어서 전송되는 것을 상기해보자. 이떄 꼬리에 붙어서 전송되는 정보가 "%" 와 "+"기호 등으로 구성된 
URL 인코딩된 데이터인 것이다. URLEncode는 영문자와 숫자 , 그리고 특수문자를 제외한 , 그리고 한글 문자는 %00
%FF 사이의 값으로 , 또한 공백은 "+"로 변환하는 메소드 이다. 



[MapPath 메소드]

MapPath 메소드는 어떤 파일에 대한 물리적인 경로를 알고자 할 경우 사용한다. 
즉,서버의 가상 경로 또는 가상 디렉토리를 실제 위치를 변환시켜주는데 이용되며,


MapPath(".") : 현제의 ASP 페이지가 있는 디렉토리를 되돌려 줌

MapPath("/") : 홈 디렉토리의 물리적 경로를 되돌려 줌

MapPath("/asp") : 홈 디렉토리의 하위 ASP 디렉토리를 되돌려 줌




[CreateObject 메소드]


서버객체의 메소드 중에서 가장 많이 사용되는 메소드이다. ASP에서는 유용하게 사용될 수 있는
컴포넌트를 사용하기 위하여 필요한 것이 바로 CreateObject이다. 이 메소드는 서버에서
동작하기 때문에 클라이언트가 별도로 ActiveX 컨트롤과 같이 다운로드 할 필요가 없다.
대부분 VB로 제작되어지는 이 서버 컴포넌트를 ASP 페이지에서 사용하고자 할 때 이용되어지는
메소드가 바로 이 CreateObject인 것이다. 아래의 코드는 ASP에서 제공하는 DAC(ADO) 컴포넌트의
인스턴스를 생성하고 사용하는 예이다.



[객체와 인스턴스]

객체는 정해진 형태의 데이터와 그 데이터에 대한 연산으로 만들어 지며, 외부에서 이 객체를 사용하려면
항상 그 객체에 데이터를 보내어 객체가 그것을 처리하도록 해야한다. 클래스로부터 생성된 각 객체를
클래스의 인스턴스라고 한다. 모든 객체는 어떤 클래스로부터 생성되었다는 의미를
강조한 말이라고 할 수 있다.
